# AGENTS.md — Blackboard

> 本文件定义 AI Agent（Claude Code、Codex CLI 等）在此项目中可执行的指令。
> 放置于项目根目录，Agent 启动时自动读取。

---

## 指令：`/sync-docs`

**用途**：根据 `instruction.md` 的规则，对项目所有文档执行一次完整的审查与同步。

### 执行步骤

Agent 收到 `/sync-docs` 后，按以下顺序执行，**每步完成后输出结果再进入下一步**：

#### Step 0 — 探测项目文档结构

在检查文件之前，先确定文档实际存放位置：

 1. 检查根目录是否有 `dev/` 子目录
 2. 检查根目录下是否直接存在 `.md` 文件

判断规则：
| 情况 | 文档根目录 |
|------|-----------|
| `dev/` 存在且包含 `.md` 文件 | `dev/` |
| `.md` 文件直接在项目根目录 | 项目根目录 |
| 两者都有 | 以 `dev/` 为主，根目录的 `.md` 单独列出 |
| 都没有 | 创建 `dev/` 目录，在其中生成文件 |

探测完成后输出：
```
📁 文档位置：<根目录 / dev/>
📄 已发现文件：X 个
```

后续所有步骤均以探测到的文档根目录为基准。

---

#### Step 1 — 扫描文档健康度

**1a. 检查文件是否存在**

首先检查以下文件是否存在，缺失的立即构建：

| 文件 | 位置 |
|------|------|
| `instruction.md` | 项目根目录 |
| `background.md` | `dev/` |
| `architecture.md` | `dev/` |
| `rule.md` | `dev/` |
| `rules/README.md` | `dev/rules/` |
| `log.md` | `dev/` |
| `todo.md` | `dev/` |
| `decisions.md` | `dev/` |
| `terms.md` | `dev/` |
| `test-sample/*.md` | `dev/test-sample/` |

缺失文件的处理规则：
- 从项目已有文档中推断项目名称、背景等信息，尽量填充模板内容
- 无法推断的字段使用 `[待补充]` 占位
- 构建前告知用户将要创建哪些文件，确认后再写入
- 构建完成后在输出中标记 `🆕`

**1b. 检查已有文件内容质量**

对存在的文件逐一检查：
- 是否存在 `[待补充]` 未填写的占位符
- 是否有空白章节（只有标题没有内容）
- 文件最后修改时间是否超过 14 天未更新（结合 `dev/log.md` 判断）

输出格式：
```
📋 文档健康检查
──────────────
🆕 terms.md          — 文件不存在，已自动创建
✅ background.md     — 完整
⚠️  architecture.md  — 第 3 节「部署架构」为空
⚠️  terms.md         — 存在 2 处 [待补充]
🕐 decisions.md      — 21 天未更新，请确认是否需要补充
```

#### Step 2 — 更新 log.md
读取自上次 `log.md` 条目日期以来，项目中发生的变化（通过 git log 或文件 mtime），生成今日条目：

```markdown
## YYYY-MM-DD — [从 git log 或文件变更自动推断标题]

### 完成
- [根据实际变更填写]

### 问题/发现
- [如有]

### 待跟进
- [如有]
```

若无 git，则提示用户描述今日工作内容后再写入。

#### Step 3 — 同步 todo.md
- 将 Step 1 中发现的 `⚠️` 和 `🕐` 问题，作为新任务追加到 `todo.md` 的「文档」分类下（去重，已存在的不重复添加）
- 检查 `todo.md` 中标记为 `[~]` 进行中的任务，询问用户是否已完成

#### Step 4 — 全文档补全建议
针对 Step 1 中发现的空白/占位内容，逐一给出补全建议：
- 展示当前空白内容的上下文
- 给出建议填写内容（基于项目其他文档的已有信息推断）
- 询问用户是否采纳，确认后写入

#### Step 5 — 输出同步报告
```
✅ /sync-docs 完成
─────────────────────────────
📝 log.md        已追加今日条目
✅ todo.md       新增 N 项文档任务
📄 补全建议      已处理 N 处占位符
⏭️  仍需人工处理  architecture.md § 部署架构（需要你补充实际环境信息）
```

## 其他约定

- Agent 在此项目中应遵守 `dev/rule.md` 中的所有开发规范
- 不确定需求时，停下来提问，不要假设
- 修改任何文件前，先展示将要做的变更，等待确认
