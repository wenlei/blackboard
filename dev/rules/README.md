# 功能块规则索引

> 每个 `.md` 文件对应一个功能模块，记录该模块内**不能从代码直接推断**的设计约定、
> 键名语义、调用顺序和禁止事项。
> AI 辅助开发时需在修改对应模块前读取相关规则文件。

## 文件列表

| 文件 | 覆盖范围 |
|------|---------|
| [config.md](config.md) | Agent 注册表、凭据管理、Config UI 弹窗行为 |

## 使用规范

- 新增功能块时，在本目录创建对应 `<module>.md`，并在此表追加一行
- 规则文件只记录**反直觉的约定**（如"凭据用 provider slug 做键而非实例名"），不重复文档注释
- 规则变更须同步更新 `dev/log.md`
