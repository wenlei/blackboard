# 测试用例：Host 策略生成与调度

> 覆盖 Host 的规则模板匹配、LLM 动态规划、分发、结果路由、中途介入

## 前置条件
- Session 已创建，配置 3 个 Agent（架构师、审查者、程序员）
- MQ Layer 正常工作
- LLM API mock 已配置

## 用例

### 2.1 首次消息 → 生成策略
| 字段 | 值 |
|------|-----|
| **输入** | 消息 `"写一个 Python 爬虫抓取 Hacker News 首页"`，Agent 角色：架构师/审查者/程序员 |
| **操作** | 用户发送消息到 inbox |
| **预期** | 规则模板匹配 → Host LLM 补充细节 → 生成多步骤策略：①架构师设计方案 → ②程序员编码 → ③审查者审查 → ④回复用户。`strategy.json` 保存 |

### 2.2 按策略分发第一步
| 字段 | 值 |
|------|-----|
| **输入** | 策略第 1 步：架构师 |
| **操作** | Host 执行分发 |
| **预期** | `dispatched` Stream 收到消息，target_agent = 架构师，内容包含原始需求 + 角色 prompt |

### 2.3 收到结果 → 路由下一步
| 字段 | 值 |
|------|-----|
| **输入** | 架构师返回设计方案（JSON 含 API 端点、数据结构、模块划分）|
| **操作** | Head 消费 `results` Stream |
| **预期** | Host 解析结果 → 参照策略路由到步骤 ② → 写 dispatched 给程序员，附上架构师的方案作为上下文 |

### 2.4 多轮协作完成
| 字段 | 值 |
|------|-----|
| **输入** | 程序员返回代码 → 审查者返回审查意见（通过）|
| **操作** | Host 完成最后一步 |
| **预期** | 策略全部步骤执行完毕 → Host 生成最终回复 → 写 outbox → `conversation.log` 包含完整多轮记录 |

### 2.5 策略中未定义的情况
| 字段 | 值 |
|------|-----|
| **输入** | Agent 返回意外结果（如程序员说"需求不明确，无法实现"）|
| **操作** | Host 评估结果 |
| **预期** | Host LLM 判断：是补充需求后重试？还是跳过？还是结束？Host 调整策略执行 |

### 2.6 用户中途介入
| 字段 | 值 |
|------|-----|
| **输入** | 用户发送 `"不用 Python 了，改用 Go"` |
| **操作** | 消息进入 inbox |
| **预期** | Host 识别为介入指令 → 重置当前步骤 → 调整策略（变成 Go 实现） → 通知程序员 Agent 重来 |

### 2.7 Agent 返回错误 — 重试机制
| 字段 | 值 |
|------|-----|
| **输入** | Agent 调用 LLM API 超时（60s 无响应）|
| **操作** | Agent 返回 error 状态 |
| **预期** | Host 收到错误 → 重试同一 Agent（最多 3 次）→ 3 次全失败 → 通知用户选择：①换 Agent 重试 ②跳过此步 ③终止 |

### 2.8 Session 结束后策略保留
| 字段 | 值 |
|------|-----|
| **输入** | session 已关闭 |
| **操作** | 查看 `GET /api/sessions/{id}/strategy` |
| **预期** | `strategy.json` 完整保留在 session 文件夹，可查看从初始消息到最终回复的完整执行路径 |
